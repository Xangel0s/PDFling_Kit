# Security Policy

## Supported Versions

Versiones activas para correcciones de seguridad:

- `0.1.x`

## Reporting a Vulnerability

Si encuentras una vulnerabilidad:

1. No la publiques en un issue publico.
2. Reportala de forma privada a los mantenedores.
3. Incluye pasos de reproduccion, impacto y evidencia.

## Response Process

Objetivos de respuesta:

- Confirmacion inicial: dentro de 72 horas.
- Evaluacion tecnica: segun severidad y reproducibilidad.
- Plan de mitigacion: correccion, release y notas de cambio.

## Security Principles

- Procesamiento local de PDF por defecto.
- Minima superficie de permisos de extension.
- Rechazo de entrada invalida y manejo de errores tipados.
- Revision de dependencias y actualizacion periodica.
